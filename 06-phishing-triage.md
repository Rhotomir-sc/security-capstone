# 06 - Phishing Triage

## Purpose

This section reviews a simple phishing email scenario in the fictional lab environment.

The goal is to identify suspicious indicators and decide how the email should be handled.

## Sample Email Scenario

A user receives an email saying that their account will be disabled soon.

The email asks the user to click a link and confirm their username and password.

## Suspicious Indicators

| Indicator | Why it looks suspicious |
|---|---|
| Urgent language | It pressures the user to act quickly |
| Unknown sender | The sender does not look trusted |
| Password request | Users should not share passwords by email |
| Suspicious link | The link should be checked before clicking |
| Generic message | The message does not look specific or personal |

## Decision

The email should be treated as suspicious.

The user should not click the link, reply to the sender, or enter credentials.

## Suggested Actions

- Report the email.
- Review the sender address and link.
- Warn users about similar phishing attempts.
- Block the sender or domain if needed.
