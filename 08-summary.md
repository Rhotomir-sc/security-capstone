# Project Summary

## Purpose

In this final section, I reviewed the main findings from the Security Capstone project.

My goal was to bring the identity, network, log, phishing, firewall, Python, and risk review sections together in one short summary.

---

## Completed Reviews

| Section | Main Result |
|---------|-------------|
| Identity Review | The `guest01` account had more access than required |
| Network Review | SSH access was open to all devices on the learning network |
| Log Review | Three failed `guest01` logins were followed by one successful login |
| Phishing Review | A fictional email used urgency and an external login link |
| Firewall Review | The SSH rule was broader than necessary |
| Python Helper | The script summarized five SSH authentication events |
| Risk Register | Four main risks were listed with recommended actions |

---

## Main Findings

- User permissions should match each role.
- SSH access should be limited to devices that need remote management.
- Repeated failed login attempts should be reviewed.
- Suspicious emails should be reported and checked through official channels.
- Small scripts can make log patterns easier to understand.
- A risk register helps connect findings with practical improvements.

---

## What I Learned

This project helped me understand how different security topics can be connected within one small learning environment.

I also practiced reviewing evidence, writing short findings, and suggesting basic security improvements.

It was created to support my learning process and student portfolio.
